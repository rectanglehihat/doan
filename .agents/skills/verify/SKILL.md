# 코드 변경 후 검증 워크플로우

CLAUDE.md에 정의된 검증 순서를 순차적으로 실행한다.

## 검증 순서

아래 순서를 **반드시 지켜서** 순차 실행한다. 앞 단계 실패 시 다음 단계 진행 전 수정한다.

### 1단계: 단위·통합 테스트

```bash
pnpm test
```

- 실패 시: 실패한 테스트 파일과 오류 메시지 보고
- 성공 시: 다음 단계 진행

### 2단계: 타입 검사

```bash
pnpm tsc --noEmit
```

- 실패 시: 타입 에러 파일과 라인 보고 (any, as, ! 사용 여부 확인)
- 성공 시: 다음 단계 진행

### 3단계: ESLint 검사

```bash
pnpm lint
```

- 실패 시: 린트 에러 파일과 규칙 보고
- 성공 시: 다음 단계 진행

### 4단계: 프로덕션 빌드 (PR 전 필수)

```bash
pnpm build
```

- 실패 시: 빌드 에러 내용 보고
- 성공 시: ✅ 모든 검증 통과

## 주의

- 빌드(`pnpm build`)는 PR 전 필수이나, 빠른 개발 중에는 1-3단계만 실행해도 됨
- 빌드 단계가 필요한지 사용자에게 확인 후 실행
