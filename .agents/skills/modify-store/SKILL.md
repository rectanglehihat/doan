# 기존 스토어/훅 수정 워크플로우

기존 Zustand 스토어(`src/store/`) 또는 커스텀 훅(`src/hooks/`)을 안전하게 수정하는 워크플로우.

## 워크플로우

### Step 1: 현황 파악 (읽기 전용)

1. 대상 파일 + 테스트 파일 읽기 (없으면 Step 2에서 신규 작성)
2. grep으로 import 사용처 확인
3. 영향 범위: 상태 타입/액션 시그니처 변경 → 사용처 전수 확인 / 반환값 추가 → 하위 호환

### Step 2: 테스트 업데이트 (test-writer 에이전트)

- 새 액션/상태 → 새 테스트 추가 (Red)
- 기존 변경 → 해당 테스트 업데이트 / 삭제 → 테스트 제거
- 스토어: `reset()`으로 각 테스트 전 초기화 확인

### Step 3: 수정 구현 (store-hook-implementer 에이전트)

- 요청된 변경만 수행, `any`/`as`/`!`/`export default` 금지
- 스토어: `reset()`이 새 필드 포함, `initialState` 업데이트, 직접 `set` 노출 금지
- 훅: 새 핸들러 `useCallback`, 새 파생 데이터 `useMemo`

### Step 4: 사용처 업데이트 (액션 시그니처 변경/반환값 제거 시)

사용처 순서대로 수정, `pnpm tsc --noEmit`으로 타입 에러 0 확인

### Step 5: 검증

`/verify` 스킬 호출 (`test → tsc → lint → build`)
