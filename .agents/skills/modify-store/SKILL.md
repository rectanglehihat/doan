# 기존 스토어/훅 수정 워크플로우

기존 Zustand 스토어(`src/store/`) 또는 커스텀 훅(`src/hooks/`)을 안전하게 수정하는 워크플로우.

## 트리거 조건

- "[스토어명]에 [액션] 추가해", "새 state 필드 추가해"
- "훅에서 [기능] 동작이 이상해", "버그 수정해줘"
- "기존 액션 시그니처 바꿔줘", "[훅명] 반환값 추가해"
- "스토어 타입 수정해", "상태 구조 변경해"

## 워크플로우

### Step 1: 현황 파악 (읽기 전용)

```
1. 대상 스토어/훅 파일 전체 읽기
2. 대응 테스트 파일 읽기 (없으면 Step 2에서 신규 작성)
3. 대상을 import하는 컴포넌트·훅 목록 grep으로 확인
4. 변경 영향 범위 정리:
   - 상태 타입 변경 → 사용처 Props/타입 파급 범위
   - 액션 시그니처 변경 → 호출부 전수 확인
   - 반환값 추가 → 사용처 영향 없음 (하위 호환)
   - 반환값 제거 → 사용처 전수 확인
```

### Step 2: 테스트 업데이트 (test-writer 에이전트)

```
test-writer 에이전트를 사용해서:
1. 기존 테스트 파일 읽고 현재 커버리지 파악
2. 수정 사항에 맞춰 테스트 추가/수정
   - 새 액션/상태 → 새 테스트 추가 (Red)
   - 기존 액션 변경 → 해당 테스트 업데이트
   - 삭제 → 해당 테스트 제거
3. 스토어라면: reset()으로 각 테스트 전 초기화 확인
4. pnpm test → 신규/변경 테스트 Red 상태 확인
```

### Step 3: 수정 구현 (store-hook-implementer 에이전트)

```
store-hook-implementer 에이전트를 사용해서:
1. 대상 파일 최소한으로 수정
2. 스토어 수정 시 반드시 확인:
   - reset()이 새 상태 필드를 포함하는지
   - initialState에 새 필드 추가됐는지
   - 직접 set 노출 없이 action 함수로만 변경 가능한지
3. 훅 수정 시 반드시 확인:
   - 새 핸들러는 useCallback 적용
   - 새 파생 데이터는 useMemo 적용
```

**수정 원칙:**
- 요청된 변경만 수행 — 관련 없는 리팩터 금지
- `any`, `as`, `!` 금지
- `export default` 금지

### Step 4: 사용처 업데이트 (필요 시)

액션 시그니처 변경 또는 반환값 제거가 있는 경우:

```
1. grep으로 확인한 사용처 파일들 순서대로 수정
2. pnpm tsc --noEmit로 타입 오류 추적
3. 타입 에러 0이 될 때까지 반복
```

### Step 5: 검증 (/verify 스킬)

`/verify` 스킬을 호출해서 전체 검증을 실행한다.

```
test → tsc → lint → build 순서로 통과 확인
```

## 유형별 체크리스트

### 스토어: 상태 필드 추가

- [ ] `DomainState` 인터페이스에 새 필드 추가
- [ ] `initialState`에 초기값 추가
- [ ] setter action 추가
- [ ] `reset()`이 새 필드를 initialState로 되돌리는지 확인
- [ ] 테스트에 새 필드 초기값 및 setter 테스트 추가

### 스토어: 상태 필드 제거 / 이름 변경

- [ ] 사용처(컴포넌트, 훅) grep으로 전부 확인
- [ ] 사용처 일괄 수정
- [ ] `initialState`, `reset()` 업데이트
- [ ] 관련 테스트 업데이트

### 스토어: 액션 시그니처 변경

- [ ] 기존 호출부 전수 확인 및 수정
- [ ] 테스트 업데이트

### 훅: 반환값 추가

- [ ] 새 값/핸들러 반환 배열/객체에 추가
- [ ] 핸들러는 useCallback, 파생값은 useMemo
- [ ] 테스트에 새 반환값 검증 추가

### 훅: 버그 수정

- [ ] 버그를 재현하는 실패 테스트 먼저 작성 (Red)
- [ ] 수정 후 통과 (Green)
- [ ] 기존 테스트 전부 통과

## 완료 기준

- [ ] 수정 대상 파일의 테스트 존재 및 통과
- [ ] 스토어라면 `reset()` action이 새 필드까지 초기화함
- [ ] `/verify` 스킬 전체 통과 (test → tsc → lint → build)
